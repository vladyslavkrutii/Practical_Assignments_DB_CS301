# Practical Assignments1 - Car Service

Я створив базу даних для станції технічного обслуговування (СТО). Проєкт відтворює роботу сервісу: облік клієнтів, їхніх автомобілів, фіксацію виконаних робіт, лист послуг та генерацію аналітичної звітності.

Таблиці
- **clients** - персональні дані клієнтів
- **cars** - інформація про транспортні засоби. Зв'язана з таблицею клієнтів зв'язком "1:many" через `foreign key (clientid)`. Поле `licenseplate` має обмеження `unique`, тому що автомобільні номери не можуть дублюватися.
- **mechanics** - реєстр всіх майстрів.
- **services** -  послуги.
- **orders** - таблиця транзакцій.

- **Запит 1 (union all)** - об'єднує виконані та скасовані замовлення в один звіт із join 5 таблиць. Я використав `union all` замість `union` для оптимізації, оскільки дані не мають дублікатів і база працює швидше.
- **Запит 2 (CTE)** - CTE рахує загальний дохід кожного майстра, `rank()` автоматично присвоює їм місця.
- **Запит 3 (group by + having)** - групує замовлення за id механіків і виводить лише тих, у кого більше ніж одне замовлення.
